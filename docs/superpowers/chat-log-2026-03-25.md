# TypeNo 项目对话记录 — 2026-03-25

## 1. 项目调研

### TypeNo 技术分析
- **项目地址**: https://github.com/marswaveai/TypeNo
- **类型**: macOS 菜单栏语音转文本应用（Swift）
- **ASR 引擎**: sherpa-onnx（通过 Coli CLI，`@marswave/coli` npm 包）
- **默认模型**: SenseVoice Small（阿里通义开源，支持中英日韩粤）
- **备选模型**: Whisper tiny.en（OpenAI 开源，仅英文）
- **处理方式**: 纯本地，无云端依赖
- **VAD**: Silero VAD
- **音频采集**: 16kHz mono

### Typeless 技术分析
- **产品**: typeless.com，AI 语音听写应用
- **创始人**: 黄松（Stanford，YC 背景）
- **ASR 引擎**: 未公开，推断为 OpenAI Whisper API（云端）
- **处理方式**: 云端两阶段（ASR + LLM 润色）
- **数据留存**: 零留存政策

### 两者对比
| | TypeNo | Typeless |
|---|---|---|
| ASR | sherpa-onnx + SenseVoice | 大概率 Whisper API |
| 处理 | 纯本地 | 云端 |
| LLM | 无 | 有 |
| 语言 | 中英日韩粤 | 100+ |

## 3. 技术方案讨论：动态词典纠正

### 三个层级
1. **热词增强（Hot Words）**: ASR 解码阶段加权，sherpa-onnx OfflineRecognizer 不支持，需后处理
2. **拼音词典后处理**: ASR 输出后用拼音匹配做同音字纠正
3. **LLM 云端纠正**: Typeless 的路线，成本高

### 最优方案（针对 TypeNo）
- 后处理纠正 + 用户自学习词典
- 存储在 `~/.typeno/hotwords.txt`
